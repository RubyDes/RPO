**Дата:** 06 декабря 2025 г.

**Статус:** Принято

**Контекст:**

Клиентское приложение должно обеспечивать удобный пользовательский интерфейс для различных типов пользователей (студенты, преподаватели, администраторы), работать на различных устройствах (десктоп, мобильные), поддерживать воспроизведение видео и интерактивные элементы тестирования.

**Рассмотренные варианты:**

1. **React:** Популярная библиотека с большим сообществом, хорошей экосистемой.
2. **Vue.js:** Прогрессивный фреймворк с мягкой кривой обучения.
3. **Angular:** Полноценный фреймворк с TypeScript из коробки, строгая архитектура.
4. **Next.js/Nuxt.js:** Фреймворки для SSR/SSG, улучшающие SEO и первоначальную загрузку.

**Решение:**

Выбран **React с TypeScript и Next.js**.

**Обоснование:**

- **Универсальность:** React подходит как для сложных административных интерфейсов, так и для пользовательских частей. (Обеспечивает PLT01, PLT02, поддержку различных ролей по SEC05)
- **TypeScript:** Обеспечивает типизацию, что снижает количество ошибок и улучшает поддержку кода. (Повышает надежность, косвенно влияя на AVA01, DUR04)
- **Next.js:** Предоставляет SSR для лучшего SEO (важно для каталога курсов) и оптимизированную загрузку.(Обеспечивает хорошую производительность PER02, улучшает доступность контента)
- **Большое сообщество:** Множество готовых компонентов (в том числе для видеоплееров, графиков аналитики). (Ускоряет разработку, LIM03)
- **PWA поддержка:** Легко создать Progressive Web App для мобильных устройств. (Обеспечивает PLT02, улучшает пользовательский опыт)

**Последствия:**

- Необходимо настроить SSR/SSG стратегию для разных страниц.
- Требуется внедрение state management (Redux Toolkit или Zustand).
- Следует использовать адаптивный дизайн и соблюдать стандарты доступности (WCAG).
